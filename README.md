## 프로젝트 개요
**프로젝트명:** 자동차 데이터 통합 시스템 : 등록·폐차 현황 및 보험 FAQ
- **프로젝트 기간:** 2025.12.10 ~ 2025.12.11 
- **프로젝트 형태:** 공공 자동차 등록·말소 통계 + 보험 FAQ 데이터를 통합한 웹 기반 대시보드(Streamlit)

**프로젝트 목표:**
  - 국가 공공데이터(자동차 등록·말소 통계)와 보험 관련 FAQ 데이터를 하나의 DB로 연계·정규화  
  - 지역·연도·차종별 자동차 등록/폐차 현황을 시각적으로 비교 분석할 수 있는 환경 제공  
  - 보험/렌터카 FAQ를 함께 조회·검색할 수 있도록 하여, 차량 이용·운행과 관련된 정보 탐색을 한 번에 처리  
  - 향후 렌터카·카셰어링/보험사 등에서 정책 수립, 고객 응대, 서비스 기획 시 참고할 수 있는 기초 데이터 플랫폼 구축
- **주요 타겟 사용자:**
  - 자동차/모빌리티 서비스 기업(렌터카, 카셰어링 등)의 데이터 분석·서비스 기획 담당자  
  - 보험사 및 관련 협회(예: 손해보험협회)의 CS·상품 기획 담당자  
  - 자동차 공공데이터를 활용하려는 데이터 분석가·연구자  
  - 자동차 등록/폐차 및 보험 민원 동향을 파악하려는 공공기관·지자체 관계자

**프로젝트의 필요성**
   * 자동차 등록 및 폐차 데이터는 매년 방대한 양이 공개되지만, 사용자들이 이를 직접 비교·이해하기 어려움.
   * 여러 기관에 흩어져 있는 데이터를 한 곳에서 통합적으로 제공
   * 신규 차량 등록 / 폐차 현황데이터 간 흐름 및 변화를 직관적으로 파악 가능
   * 보험 관련 FAQ 제공으로 사용자 편의성 증대
   * 기업, 기관 연구자에게 정책·시장 분석 자료로 활용 가능

## 프로젝트 소개

우리나라에서는 매년 신규 자동차 등록 대수가 폐차·말소 대수보다 훨씬 많아 도로 위 차량 수가 지속적으로 증가
그만큼 자동차 보험 가입·변경, 보상 문의 등 보험 관련 정보에 대한 관심과 수요도 함께 증가

이 프로젝트는 공공 **자동차 등록·말소 통계**와 **자동차 보험 FAQ**를 하나의 플랫폼에서 통합해 보여주는 웹 대시보드  
Streamlit 기반 화면에서 연도·지역·차종별 등록/폐차 현황을 비교하고, 동시에 자동차 보험 FAQ를 검색·조회함으로써 차량 증가 추세와 보험 관련 관심도를 함께 파악할 수 있도록 하는 것을 목표

## 트러블슈팅
### 문제 개요
- 문제 상황 1: 다수 팀원의 파일 수정 충돌 문제
> 여러 팀원이 각 페이지에 데이터를 연결하기 위해 동일한 파일을 동시에 수정해야 하는 상황이 발생
> 이로 인해 Git 충돌 및 코드 덮어쓰기와 같은 협업 문제가 발생할 가능성이 높다고 판단

- 발생 배경
> 전체 기능(Tab) 관련 로직이 하나의 파일에 집중되어 있어, 각 기능을 수정할 때마다 동일한 파일을 반복적으로 수정해야 하는 구조
> 이러한 구조로 인해 협업 시 파일 충돌 문제가 자주 발생할 수 있는 환경이 형성

### 오류 코드 메세지
> 실제 오류 코드나 메시지가 출력되지는 않았으나, 여러 팀원이 동일한 파일을 수정하는 과정에서 Git 충돌, 코드 덮어쓰기, 버전 관리 오류 등이 발생할 가능성이 높다고 판단

### 원인 분석
> 기능이 하나의 파일에 집중되어 있어 기능별 독립성이 확보되지 않은 구조
> 여러 팀원이 동일한 코드 영역을 동시에 수정하면서 충돌이 발생할 가능성이 상승
> 파일이 구조적으로 분리되어 있지 않아 협업 효율이 떨어지고 유지보수가 어려운 문제가 존재

### 해결책
> 전체 구조를 Tab 단위로 파일을 분리하여 기능별로 독립적인 관리가 가능하도록 재설계
> 최상단 실행 파일(main)을 생성하여 각 Tab 파일을 불러오는 구조로 변경함으로써 협업 시 충돌 가능성을 최소화
> 팀원별로 담당 Tab을 명확히 분리하여 동일 파일을 동시에 수정하지 않도록 역할을 재정의
> 파일 분리 이후 협업 과정에서 충돌이 크게 감소하여 안정적으로 개발을 진행
